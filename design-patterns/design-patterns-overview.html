Design Pattern Overview

  • Design Patterns were popularised by the book "Design Patterns: Elements of Reusable Object Oriented Software"
  • The book describes 23 useful patters that you can implement to solve common programming problems
  • These patterns are not algorithms or specific implementations
  • Design patterns are like ideas, opinions and abstractions that can be useful in certain situations to solve a particlar problem
  • Design patterns are classified into three main catergories: creational, structural and behavioural patterns
